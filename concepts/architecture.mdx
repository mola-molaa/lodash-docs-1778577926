开发者在使用时，可以根据需要<Mark bold>仅引入所需模块</Mark>，例如通过如下方式：

```js
import { chunk, uniq } from "lodash-es";
```

这种结构既保证了功能完整性，又提升了<Mark color="green">可维护性与可扩展性</Mark>。

## 构建系统

Lodash 提供了两种主要构建产物，分别面向不同使用场景：

### main 构建

main 构建是<Mark bold>默认、最常用</Mark>的构建方式，它生成的模块保持传统命令式风格，方法签名与参数顺序更贴近大多数开发者的直觉。

特点包括：

<BulletedList>
    支持完整功能集，包含 lodash 全部方法
</BulletedList>
<BulletedList>
    方法参数顺序以数据在前、配置在后为主
</BulletedList>
<BulletedList>
    适合绝大多数业务项目与工具库
</BulletedList>

对应入口与产物通常被发布为：

<Paragraph textAlign="center">
    <Mark bold>lodash</Mark> / <Mark bold>lodash-es</Mark>
</Paragraph>

### fp 构建

fp 构建是<Mark bold>函数式编程友好</Mark>的构建版本，它对所有方法进行了<Mark italic>自动柯里化</Mark>与参数顺序调整，使“数据”位于参数列表最后，更适合函数组合。

其核心特性如下：

<BulletedList>
    所有方法默认柯里化，支持部分参数应用
</BulletedList>
<BulletedList>
    参数顺序统一为：配置参数在前，数据参数在后
</BulletedList>
<BulletedList>
    更方便与 compose / flow 等函数组合工具配合使用
</BulletedList>

如果你希望以函数式风格组织代码，可以优先选择 fp 版本：

```js
import { map, filter, compose } from "lodash/fp";
```

关于 fp 构建的更多细节，可以参考<Link href="/concepts/fp">函数式编程（fp）概念</Link>页面。

## lodash-cli 工具

lodash-cli 是 Lodash 的<Mark bold>内部构建与定制工具</Mark>，它负责：

<NumberedList>
    读取源码目录，解析模块依赖关系
</NumberedList>
<NumberedList>
    根据配置生成 main 或 fp 构建产物
</NumberedList>
<NumberedList>
    支持按模块、按方法生成<Mark italic>自定义子集构建</Mark>，减少最终包体积
</NumberedList>
<NumberedList>
    生成不同模块格式（如 CommonJS、ES Module、UMD）的发布文件
</NumberedList>

通过 lodash-cli，Lodash 能够在保证功能一致性的前提下，同时满足多种运行环境与使用习惯。
