欢迎参与 Lodash 项目的贡献！本文档将指导你完成从环境搭建到提交 Pull Request 的完整流程，帮助你快速融入项目开发。

## 开发环境搭建

在开始贡献代码前，需要先完成本地开发环境的准备：

<NumberedList>
    确保你的机器已经安装 Node.js（建议版本 12 及以上）和 npm 包管理工具。
</NumberedList>
<NumberedList>
    从 GitHub 上 fork Lodash 的官方仓库到你的个人账号下。
</NumberedList>
<NumberedList>
    将你 fork 后的仓库克隆到本地：
    <Paragraph>
        执行命令 <Mark bold>git clone https://github.com/你的用户名/lodash.git</Mark>，将仓库拉取到本地目录。
    </Paragraph>
</NumberedList>
<NumberedList>
    进入项目目录并安装依赖：
    <Paragraph>
        执行 <Mark bold>cd lodash</Mark> 进入项目目录，然后执行 <Mark bold>npm install</Mark> 安装项目所需的依赖包。
    </Paragraph>
</NumberedList>
<NumberedList>
    添加上游仓库关联，方便后续同步官方最新代码：
    <Paragraph>
        执行 <Mark bold>git remote add upstream https://github.com/lodash/lodash.git</Mark> 关联官方仓库。
    </Paragraph>
</NumberedList>

## 代码风格要求

Lodash 项目有统一的代码风格规范，所有提交的代码必须符合以下要求：

<BulletedList>
    遵循项目既有的 JavaScript 编码风格，保持代码缩进、空格、命名等格式一致。
</BulletedList>
<BulletedList>
    函数命名采用驼峰式命名法，语义清晰，能够准确表达函数的功能。
</BulletedList>
<BulletedList>
    新增函数需要添加完整的 JSDoc 注释，说明函数的用途、参数、返回值和使用示例。
</BulletedList>
<BulletedList>
    避免引入不必要的依赖，保持 Lodash 轻量化的特性。
</BulletedList>
<BulletedList>
    代码提交前需要通过项目自带的代码检查工具校验，确保没有格式问题。
</BulletedList>

## 测试运行

所有新增或修改的代码都需要配套对应的测试用例，并且保证所有测试通过：

<NumberedList>
    运行全量测试：
    <Paragraph>
        在项目根目录执行 <Mark bold>npm test</Mark> 命令，运行所有测试用例，确保没有失败项。
    </Paragraph>
</NumberedList>
<NumberedList>
    运行单个模块测试：
    <Paragraph>
        如果需要测试特定模块，可以执行 <Mark bold>npm test -- --grep "模块名"</Mark> 命令，只运行对应模块的测试用例。
    </Paragraph>
</NumberedList>
<NumberedList>
    新增功能必须添加对应的测试用例，覆盖正常场景、边界场景和异常场景。
</NumberedList>
<NumberedList>
    修改现有功能时，需要同步更新对应的测试用例，确保修改后的功能符合预期。
</NumberedList>

## Pull Request 流程

当你完成代码开发和测试后，就可以提交 Pull Request 到 Lodash 官方仓库：

<NumberedList>
    同步上游仓库的最新代码到你的 fork 仓库：
    <Paragraph>
        执行 <Mark bold>git fetch upstream</Mark> 拉取官方最新代码，然后执行 <Mark bold>git merge upstream/main</Mark> 合并到你的本地分支。
    </Paragraph>
</NumberedList>
<NumberedList>
    基于最新的 main 分支创建新的开发分支：
    <Paragraph>
        执行 <Mark bold>git checkout -b feat/你的功能名称</Mark> 创建专属开发分支，避免直接在 main 分支上修改。
    </Paragraph>
</NumberedList>
<NumberedList>
    将你的修改提交到本地仓库，提交信息需要清晰说明本次修改的内容，遵循约定式提交规范。
</NumberedList>
<NumberedList>
    将你的开发分支推送到你 fork 的远程仓库：
    <Paragraph>
        执行 <Mark bold>git push origin feat/你的功能名称</Mark> 推送代码。
    </Paragraph>
</NumberedList>
<NumberedList>
    打开 GitHub 上你 fork 的仓库页面，点击 <Mark bold>Compare & pull request</Mark> 按钮，创建到 Lodash 官方仓库 main 分支的 Pull Request。
</NumberedList>
<NumberedList>
    Pull Request 的描述需要包含修改的背景、实现思路和测试情况，方便维护者快速 review。
</NumberedList>
<NumberedList>
    根据维护者的 review 意见修改代码，修改后推送更新到同一个 Pull Request 即可，不需要新建。
</NumberedList>
