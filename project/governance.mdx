Lodash 作为一个被广泛使用的 JavaScript 工具库，拥有清晰、透明的治理结构，以保障项目的长期稳定演进、安全响应与高质量发布。本文档介绍 Lodash 的核心治理角色与团队组成。

## 技术指导委员会（TSC）

技术指导委员会（Technical Steering Committee，TSC）是 Lodash 项目的最高技术决策机构，负责：

<BulletedList>
    制定项目整体技术方向与长期路线图
</BulletedList>
<BulletedList>
    评审并决策重大架构变更、API 设计调整与破坏性更新
</BulletedList>
<BulletedList>
    仲裁跨团队技术分歧，确保设计一致性与可维护性
</BulletedList>
<BulletedList>
    维护核心代码库，并监督发布流程与版本策略
</BulletedList>
<BulletedList>
    任命或调整安全分流团队与发布团队的成员
</BulletedList>

TSC 成员通常由长期活跃的核心贡献者、主要维护者构成，其变更会通过项目公告进行公开说明。

## 安全分流团队

安全分流团队负责 Lodash 项目的安全问题接收、评估与响应，主要职责包括：

<BulletedList>
    接收并初步评估来自社区或安全研究者的安全报告
</BulletedList>
<BulletedList>
    对安全问题的严重程度进行分类与优先级排序
</BulletedList>
<BulletedList>
    协调修复方案，并安排补丁在受影响的版本中落地
</BulletedList>
<BulletedList>
    组织安全公告发布，确保用户及时获得风险信息与升级建议
</BulletedList>

安全分流团队会与 TSC 保持密切协作，在涉及核心 API 或架构层面的安全修复时，需获得 TSC 的技术确认。

## 发布团队

发布团队负责 Lodash 版本的具体构建、测试与发布工作，其职责包括：

<BulletedList>
    维护 CI/CD 流程，保障自动化测试、打包与发布链路的稳定性
</BulletedList>
<BulletedList>
    执行版本发布流程，包括版本号管理、变更日志整理与 npm 发布
</BulletedList>
<BulletedList>
    验证发布产物在不同环境下的兼容性，降低发布风险
</BulletedList>
<BulletedList>
    配合安全分流团队，优先完成安全修复版本的发布
</BulletedList>

发布团队成员需遵循既定的发布规范，并在必要时向 TSC 汇报发布计划与异常情况。
